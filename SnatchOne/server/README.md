# 🚀 Snatch Unified Server v3.0

Объединенный сервер для системы Snatch, включающий админ-панель, API бота и систему лицензий.

## 📋 Возможности

### 1. **Admin Panel** (Админ-панель)
- Управление пользователями и ролями (SuperAdmin, Admin, Team, Translator)
- Генерация и управление лицензионными ключами
- Мониторинг активных сессий в реальном времени
- Статистика по отправленным сообщениям и письмам
- Управление профилями операторов

### 2. **Bot API** (API бота)
- WebSocket сервер для расширения Chrome
- Автоматическая рассылка инвайтов и писем
- Система ротации сообщений
- Поддержка триггеров (Like, Wink, View и т.д.)
- Прикрепление медиафайлов

### 3. **License System** (Система лицензий)
- Проверка лицензионных ключей
- Автоматическое создание аккаунтов переводчиков
- Управление сроками действия лицензий
- Привязка лицензий к пользователям

## 🔧 Установка

### 1. Установите зависимости:
```bash
npm install
```

### 2. Запустите сервер:
```bash
npm start
```

Или для разработки с автоперезагрузкой:
```bash
npm run dev
```

## 🌐 Доступ

После запуска сервер будет доступен по адресу:

- **Admin Panel**: http://localhost:3000/admin
- **Bot API**: ws://localhost:3000
- **License Check**: http://localhost:3000/check

### Данные для входа в админку:
- **Логин**: `admin`
- **Пароль**: `admin123`

## 📊 Оптимизации для 100+ пользователей

Сервер оптимизирован для работы с большим количеством одновременных подключений:

### 1. **Управление памятью**
- Автоматическая очистка старых сессий (каждый час)
- Ограничение истории до 1000 записей на сессию
- Удаление неактивных сессий старше 24 часов

### 2. **Производительность**
- Использование SQLite для хранения данных
- Эффективное управление WebSocket соединениями
- Оптимизированные запросы к базе данных

### 3. **Масштабируемость**
- Поддержка до 100+ одновременных WebSocket подключений
- Изолированные сессии для каждого пользователя
- Независимая обработка запросов

## 🗂️ Структура базы данных

### Таблица `users`
- `id` - ID пользователя
- `username` - Логин
- `password` - Хешированный пароль
- `role` - Роль (superadmin, admin, team, translator)
- `nickname` - Отображаемое имя
- `avatar` - URL аватара
- `telegram` - Telegram контакт

### Таблица `licenses`
- `key_hash` - Хеш лицензионного ключа
- `exp_time_ms` - Время истечения (миллисекунды)
- `note` - Описание/имя оператора
- `creator_id` - ID создателя лицензии
- `translator_id` - ID переводчика (владельца)
- `max_uses` - Максимальное количество использований
- `current_uses` - Текущее количество использований

## 🔐 Роли пользователей

### SuperAdmin
- Полный доступ ко всем функциям
- Управление всеми пользователями
- Генерация лицензий для любого пользователя
- Принудительная смена паролей
- Удаление пользователей

### Admin
- Управление лицензиями
- Просмотр всех операторов
- Генерация лицензий
- Продление лицензий

### Team
- Генерация лицензий для своей команды
- Просмотр только своих лицензий
- Управление своими операторами

### Translator
- Просмотр только своей лицензии
- Обновление своего профиля
- Работа с ботом через расширение

## 📡 API Endpoints

### Admin API
- `POST /admin/api/login` - Вход в систему
- `GET /admin/api/status` - Получение статистики и данных
- `POST /admin/api/generate` - Генерация лицензий
- `POST /admin/api/revoke` - Отзыв лицензии
- `POST /admin/api/extend` - Продление лицензии
- `POST /admin/api/profile` - Обновление профиля
- `POST /admin/api/users` - Создание пользователя
- `POST /admin/api/delete_user` - Удаление пользователя
- `POST /admin/api/force_password` - Смена пароля пользователя
- `GET /admin/api/all_users` - Получение всех пользователей

### Bot API
- `POST /` - Загрузка настроек бота
- `POST /check` - Проверка лицензии
- `WebSocket /` - WebSocket соединение для бота

## 🔄 Миграция с предыдущих версий

Если вы переходите с раздельных серверов (`admin_site` и `local_server`):

1. Скопируйте базу данных `database.sqlite` в папку `server/`
2. Обновите настройки расширения, указав новый URL: `http://localhost:3000`
3. Остановите старые серверы
4. Запустите новый unified server

## 🛠️ Переменные окружения

Вы можете настроить сервер через переменные окружения:

```bash
PORT=3000                    # Порт сервера
HOST=0.0.0.0                # Хост для прослушивания
JWT_SECRET=your_secret_key  # Секретный ключ для JWT
PUBLIC_URL=http://your-domain.com  # Публичный URL сервера
```

## 🐛 Отладка

Для отладки проблем:

1. Проверьте логи сервера в консоли
2. Убедитесь, что база данных `database.sqlite` существует
3. Проверьте, что порт 3000 не занят другим приложением
4. Убедитесь, что расширение использует правильный URL сервера

## 📞 Поддержка

При возникновении проблем проверьте:
- Версию Node.js (рекомендуется 16+)
